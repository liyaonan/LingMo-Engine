## 五、伤害公式

### 5.1 基础伤害

```
base_damage = atk² × power / (atk + effective_def × R)    # R = 6
effective_def = def × (1 - def_ignore)
final = base × tag_mult × element_mult × variance × crit_mult
```

### 5.2 公式性质

- **同境界 50 vs 50**：约 **7 回合** 互相击杀
- **跨境界**：高境界 atk 碾压低境界 def，自然形成一击秒杀
- **power 范围**：0.6（普攻级，base）~ 3.0（神通级，cap）

### 5.3 暴击

| 条件 | 倍率 |
|------|------|
| 普通暴击 | **×1.5** |
| 标签匹配时暴击 | **×2.0** |

暴击率 = 5%（基础）+ 道路 crit_mod + primary_attr / 500，范围 [5%, 60%]

### 5.4 命中率

```
命中率 = 75% + (caster_spd - target_spd) / max(caster_spd, 1) × 20% + path_hit_mod
```

范围 [40%, 99%]

### 5.5 逃跑率

```
逃跑率 = 40% + (actor_order - enemy_order) × 12% + (actor_spd / max(enemy_spd, 1) - 1) × 15%
```

范围 [5%, 90%]

### 5.6 五行生克

| 关系 | 倍率 |
|------|------|
| 克（火→金） | **×1.30** |
| 被克（水→土） | **×0.70** |
| 相生（木→火） | **×0.80** |
| 同属性 | **×1.10** |
| 无关 | ×1.00 |

相生链：木→火→土→金→水→木

### 5.7 方差

所有伤害 × `random(0.85, 1.15)`

### 5.8 等级差衰减（双侧）

技能携带 `level` 字段（生成时等级）。伤害公式引入双侧衰减：

```
衰减系数 = max(0.05, (skill_cm / target_cm) ^ 0.25)
```

衰减应用于 power、modifier、chance、ratio（乘法）和 value、count（clamp ≥ 0）。

| 技能Lv | 对方Lv | 衰减系数 |
|--------|-------|---------|
| 同级 | — | 1.00 |
| 12 | 13 | 0.93 |
| 9 | 13 | 0.66 |
| 6 | 13 | 0.23 |
| 3 | 13 | 0.10 |
| 0 | 13 | 0.08 |

双侧叠加：Lv0 技能在 Lv13 对 Lv13 场景下，最终伤害 = 原始 × 0.08 × 0.08 = ×0.0064

| shield | value=15 | +5 | 9999 | 2 | ✓ | max(3, floor(√value×2+dur×3)) | max(2, dur+1) |
| dispel | count=1 | +1 | 5 | 2 | — | max(5, count×10) | max(2, count) |
| lifesteal | p=0.4,r=0.05 | p+0.05,r+0.01 | p=4.0,r=0.5 | 3 | — | max(2, floor(power×3)) | max(1, floor(power)-1) |

dot / fixed_dot 默认持续 3 回合（default_duration: 3）。

## 十二、概率驱动生成流程

```
1. 计算预算 = budget_table[level] × rarity_multiplier
2. 过滤有效槽位（type 必须在 templates 中）
3. 互斥过滤（exclusions 规则：同组保留 weight 最高者）
4. 归一化 effect_slots 的 weight
5. 循环抽签直到预算耗尽：
   a. 按 weight 加权随机选 slot
   b. 预算不够 → 停
   c. 检查是否叠满 cap → 已满则浪费
   d. 首次抽中赋予 base，后续叠加 stack_increment，clamp 到 cap
6. 合并同类效果（按 type+target+stat+element+mode 分组）
7. 应用 value_scale（仅 value_scale: true 的效果）
8. 计算 auto_cost × cost_scale
9. 计算 auto_cooldown
```
